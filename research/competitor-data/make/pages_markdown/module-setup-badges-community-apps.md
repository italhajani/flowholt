# Module setup badges, community apps

3 min

## Improvements and changes

* Using templates to create.scenarios﻿ is now much easier and more flexible.

* We added an "X" icon that allows you to easily close the module settings. This means you can now open and set up modules in any order you prefer.

![Document image](https://images.archbee.com/4CkrlJIBl1di_p1x71ery-mIlCUtTLjgtF9D9GPkEAH-20250228-122909.png?format=webp "Document image")

﻿

* We introduced badges for each module to make sure you don't miss any important information:

| Document image  ﻿ | A green badge indicates that all required fields are filled in. |
| --- | --- |
| Document image  ﻿ | An orange badge signals that you need to provide some additional information. |
| Document image  ﻿ | A gray badge reminds you that you still need to get started with setting up the module. |

Once you complete the setup and all your badges turn green, you will see a side panel that gives you the option to run, edit, or schedule your scenario﻿.

![Document image](https://images.archbee.com/4CkrlJIBl1di_p1x71ery-S4xwJEETbdjVDP_rpied3-20250228-131019.png?format=webp "Document image")

﻿

* We renamed the **add-on apps** to **community apps**. In addition, on the Make﻿ [integrations](https://www.make.com/en/integrations?addOnApps=1&nativeApps=0 "integrations") page, we show the number of scenarios﻿ that are using the community app.

![Document image](https://images.archbee.com/4CkrlJIBl1di_p1x71ery-fhGQldZtiOqqqrPyyikvF-20250228-131137.png?format=webp "Document image")

﻿

* On the Make﻿ [integrations](https://www.make.com/en/integrations?addOnApps=1&nativeApps=0 "integrations") page, we show the verification badge for Make﻿ verified apps with their author and maintainer. This brings more clarity to who developed the app and who you should reach out to if you experience issues.

![Document image](https://images.archbee.com/4CkrlJIBl1di_p1x71ery-9cX8H8CUQL9POlSwNk04x-20250228-131228.png?format=webp "Document image")

﻿

## Fixed issues

* If a scenario﻿ contained an iterator followed by an aggregator, sometimes it would get stuck when processing a large number of bundles. The scenario﻿ then failed to create scenario﻿ logs. Now scenarios﻿ run smoothly and create scenario﻿ logs.
