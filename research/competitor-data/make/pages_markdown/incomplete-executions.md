# Incomplete executions

1 min

Incomplete executions are a safety feature that protects your scenarios﻿ from stopping due to errors and from data loss that could happen. When a scenario﻿ encounters an error with incomplete executions enabled, Make﻿ stores the unfinished scenario run in the **Incomplete executions** tab.

Incomplete executions are disabled by default. To store incomplete executions, enable the **Store** **incomplete executions** option in the **Scenario** **settings**.

![Document image](https://images.archbee.com/oAyFj2GHlBeBVWF5OAir2/JITv5bFhH08k053Dq2pdC-20260204-150539.png?format=webp "Document image")

﻿

You can handle incomplete executions in the following ways:

* let Make﻿ retry the incomplete executions automatically for the [supported error types](/automatic-retry-of-incomplete-executions)﻿ or use the **Break** error handler

* ﻿[resolve](/manage-incomplete-executions#resolve-an-incomplete-execution)﻿ the incomplete executions manually

* ﻿[delete](/manage-incomplete-executions#delete-incomplete-executions)﻿ the incomplete executions

The maximum number of incomplete executions from all scenarios﻿ in all teams in an organization depends on your usage allowance. If the limit is exceeded, you will receive an error message.
