# FlowHolt UI Direction

## Current truth

The current Studio canvas is functionally useful, but visually not acceptable as a final product direction.
It is only a foundation.

## Design standard to target

FlowHolt Studio should feel:
- modern
- neat
- clean
- premium
- calm
- clear
- structured
- slightly classical rather than flashy

## Reference direction

The visual quality target is closer to polished workflow products like the shared references:
- clean canvas
- intentional spacing
- elegant node cards
- subtle controls
- professional toolbar chrome
- balanced light and dark treatment where appropriate
- clearer hierarchy between canvas, side controls, and execution tools

## Product language rule

User-facing UI should stay simple.
Avoid technical system words in visible navigation and headings when possible.
Prefer labels like:
- Create
- Chat
- Studio
- Run
- Save
- Steps
- Tools

## Build priority rule

From this point, Studio UI polish is a first-class priority.
Do not treat design as secondary after functionality.
When choosing between more backend breadth and a much better Studio experience, prioritize the Studio experience first.

## Immediate next UI goals

1. Replace the rough default React Flow look with a custom-designed canvas.
2. Design custom workflow nodes with stronger hierarchy, icon areas, labels, and metadata.
3. Create a more polished top toolbar and left tool rail.
4. Make the canvas feel intentional through spacing, shadows, controls, and subtle surfaces.
5. Keep the interface readable and professional, not toy-like.
